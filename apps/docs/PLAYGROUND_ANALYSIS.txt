# Laporan Analisis Sistem Playground & Demo

## Pendahuluan
Saya telah melakukan audit menyeluruh terhadap sistem playground, termasuk `playground.tsx`, `demo-loader.tsx`, `auto-controls.tsx`, `code-generator.ts`, dan mekanisme loading file di `src/lib/source-fetcher.ts` serta API route terkait.

Berikut adalah hasil temuan saya, mulai dari celah keamanan hingga kualitas kode yang... yah, perlu dibicarakan.

## 🚨 Temuan Keamanan (Security Vulnerabilities)

### 1. Arbitrary File Read (Directory Traversal) di Mode Development
**Lokasi:** `apps/docs/src/lib/source-fetcher.ts`
**Severity:** ⚠️ **High** (hanya di Dev Mode, tapi tetap berbahaya)

Di fungsi `fetchComponentSource`, path file dibangun langsung dari input user (`slug`) tanpa validasi yang memadai:

```typescript
// src/lib/source-fetcher.ts
export async function fetchComponentSource(slug: string[]): Promise<SourceResponse> {
  // ❌ BAHAYA: Menggabungkan input user langsung ke path sistem
  const componentPath = path.join(SRC_ROOT, slug.join("/"));
  
  // ... kode membaca file ...
  await scan(entryFile, 0);
}
```

Meskipun `scan` memiliki pengecekan `startsWith(SRC_ROOT)` untuk *recursive imports*, file entry point (`entryFile`) **TIDAK DICEK**.
Jika attacker (atau developer yang iseng) bisa memanipulasi `slug` untuk memuat `..`, mereka bisa membaca file apa saja di sistem Anda saat mode development aktif.

**Saran Perbaikan:**
Selalu validasi bahwa `path.resolve(componentPath)` masih berada di dalam `SRC_ROOT` sebelum membaca file.

```typescript
const resolvedPath = path.resolve(SRC_ROOT, slug.join("/"));
if (!resolvedPath.startsWith(SRC_ROOT)) {
  throw new Error("Invalid path: Stay in the sandbox, kid.");
}
```

## 🔥 Roasting & Code Quality Analysis

### 1. Parsing Code Pakai Regex? Serius? 🤦‍♂️
**Lokasi:** `apps/docs/src/components/app/playground/code-generator.ts`

Saya melihat Anda mencoba memodifikasi source code TypeScript menggunakan Regular Expressions.
```typescript
const funcRegex = new RegExp(
  `export default function ${componentName}\\(\\{[\\s\\S]*?\\}(?::\\s*\\w+)?\\)\\s*\\{`
);
```

Ini adalah dosa besar dalam dunia engineering. Regex itu rapuh (brittle).
- Ganti spasi dikit? **Break.**
- Ganti format props? **Break.**
- Ada komentar di tengah props? **Break.**

Anda sedang membangun parser compiler dengan selotip dan lem kertas. Ini bukan cara yang scalable atau robust.

**Saran:** Gunakan AST (Abstract Syntax Tree) parser seperti `ts-morph` atau `babel`. Jauh lebih aman, lebih akurat, dan Anda tidak perlu berdoa setiap kali ada perubahan format code.

### 2. "Compiler" Manual yang Menyedihkan
Di file yang sama, Anda melakukan loop manual untuk mengganti props:

```typescript
for (const [key, value] of Object.entries(controlValues)) {
  // ... logika string replace yang rumit ...
  code = code.replace(propLineRegex, `\n      ${key}="${value}"`);
}
```

Ini sangat menyakitkan untuk dibaca. Anda mencoba meniru logika React rendering tapi dengan string manipulation. Edge case-nya jutaan. Bagaimana jika prop-nya object? Array? Function? Logika ini hanya akan bekerja untuk tipe data primitif sederhana dan akan hancur begitu ada kompleksitas sedikit saja.

### 3. Hardcoded Paths di Mana-mana
**Lokasi:** `apps/docs/src/components/app/playground/use-demo-loader.ts`

```typescript
const mod = await import(`@/components/examples/${section}/${demo}`);
```

Struktur folder Anda terkunci mati. Jika Anda ingin memindahkan folder `examples`, Anda harus refactor ulang semua logic ini. Tidak fleksibel.

## Kesimpulan & Saran

Sistem ini "berfungsi" untuk kasus sederhana, tapi fondasinya rapuh seperti rumah kartu.

1.  **Fix Security Hole:** Tambahkan validasi path di `source-fetcher.ts` SEKARANG JUGA.
2.  **Buang Regex:** Rewrite `code-generator.ts` menggunakan `ts-morph` atau library AST lainnya. Jangan keras kepala pakai regex.
3.  **Refactor Demo System:** Pertimbangkan arsitektur yang lebih deklaratif daripada mencoba mem-parse file demo secara runtime. Mungkin generate metadata saat build time?

Sekian analisisnya. Jangan baper, ini demi kebaikan codebase Anda. 🚀
